문제점

맵을 구성시 변수를 하나하나 입력하는 것은 너무 시간이 오래 걸리고 가독성이 나빠서 입력 후 제가 원하지 않는 엉뚱한 값들이 나오는 경우가 대부분이었습니다.


해결 방안

Editor Customize를 통한 Inspector를 변경하여 맵 구현 효율성 및 가독성을 높혔습니다.
옆 사진과 같이 장애물이 필요한 곳에 부분에 클릭을 하고 타입을 일괄적용이 가능하도록
구현하였습니다.

또한 일괄 적용 버튼을 누를 때 모든 체크한 위치에 동일한 장애물 값을 넣는게 아닌
디폴트 값인 0인 부분에만 적용되게 구현하여

여러가지 값들을 더 편리하게 입력이 가능할 뿐만 아니라
차후 장애물 추가시 새로운 값을 입력만 하면 되는 형태로 범용성 높게 구현하였습니다.

문제점

데이터 저장 방식 playerprefs 사용시 데이터의 삭제 및, 앱 삭제 후 다시 앱을 설치 했을 때 기존에 했던 데이터가 삭제가 되지 않고 그대로 유지된다는
문제가 생겼습니다.


해결 방안

이를 해결하기 위해 JSON형태로 데이터를 저장하는 방식을 사용하였습니다.
이 저장방식을 이용하여 각 맵의 정보, 유저의 데이터를 저장하는데 사용하였습니다.

이 방식을 통하여 새로운 스테이지를 만들때 마다 Scene을 생성하는 것이 아닌
각각 맵에 대한 데이터를 저장 -> 게임 씬에서 데이터를 불러와서 맵을 구성한다.
라는 방식으로 구현하였습니다.

이를 통하여 현재 앱에서 사용된 Scene의 갯수는 시작, 맵 선택, 게임 화면
이렇게 3가지 입니다.
